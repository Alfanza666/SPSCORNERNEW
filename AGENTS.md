# Guidelines for AI Agents

## Changelog
To ensure traceability for the user, every time a change is made to the codebase (such as bug fixes, new features, etc.), you **MUST** add a changelog entry to the `changelog.txt` file.

Use the following markdown format:

```markdown
## [Version/Update] — [Short title of the change]
**Date: YYYY-MM-DD**

### Root Cause / Description:
[Explain the problem or the reason for the change]

### Fix / Changes:
- [Point 1]
- [Point 2]
```

## Language
The user prefers communicating and keeping logs in Indonesian (`changelog.txt` is in Indonesian). Write the changelog in Indonesian where appropriate.
