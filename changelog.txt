## v4.5.17 — CRITICAL FIX: ChunkLoadError "Failed to fetch dynamically imported module"
**Tanggal: 2026-05-05**

### Root Cause:
Setiap kali ada deploy baru ke Vercel, Vite generate JS chunk files dengan **hash baru** (misal `Cart-n6-dpCxa.js` → `Cart-BARU123.js`). User yang browser-nya masih cache `index.html` lama akan mencoba load chunk lama yang sudah tidak ada → **404 → Error "Failed to fetch dynamically imported module"**. Ini diperparah oleh Service Worker yang bisa cache `index.html` lama lebih lama.

### Bug yang Diperbaiki:
- **[CRITICAL 🔴] `App.tsx` — Semua `React.lazy()` tidak punya retry:** Saat chunk gagal dimuat, langsung crash ke ErrorBoundary tanpa mencoba ulang. **FIXED** → Dibuat helper `lazyWithRetry()` yang retry 3x dengan jeda 1-3 detik sebelum give up.
- **[CRITICAL 🔴] ErrorBoundary — Tombol "Coba Lagi" tidak efektif:** `resetErrorBoundary()` hanya re-render React tree, tidak bisa memuat chunk yang sudah 404. **FIXED** → Tombol sekarang memanggil `window.location.reload()` (hard reload) agar browser fetch `index.html` terbaru.
- **[FIX] Auto-reload untuk ChunkLoadError:** `ErrorFallback` sekarang mendeteksi ChunkLoadError dan **otomatis reload halaman sekali** (dengan flag `sessionStorage` untuk mencegah infinite reload loop).
- **[UX] Pesan error lebih manusiawi:** Untuk ChunkLoadError, judul berubah menjadi "Versi Baru Tersedia" dengan pesan "Halaman akan dimuat ulang otomatis" — bukan lagi pesan teknis yang membingungkan user.

### Catatan QA:
- **Mengapa ini terlewat?** Error ini hanya terjadi di production setelah deployment — tidak bisa direproduksi di `localhost` karena chunk tidak pernah di-invalidate saat dev server berjalan. Ini adalah "deployment-time bug" yang memerlukan monitoring production aktif.
- **Pelajaran:** Setiap implementasi `React.lazy()` WAJIB disertai `lazyWithRetry()` atau mekanisme serupa sejak awal.
